github notları

temel terimler

  1. repository : Kısa İsmiyle repo kodarımızın saklandığı yerdir
  2. master : depodaki ana proje sürümüdür
  3. branch : Git ağaç mantığı ile çalışır ve projedeki geliştirmelerimizi ağaç dalı mantığı ile ayrı dallarda saklayabilir, sonrasında master branch de birleştirebiliriz
  4. merge : branchlerde yapılan işlemlerin masterda birleştirilmesidir
  5. push : dosyalarımızı depoya göndermemizi sağlar
  6. pull : depodaki dosyalarımızı çekmemizi sağlar
  7. .gitignore : git in takip etmesini istemediğimiz dosya ve klasörleri belirttiğimiz dosyadır
  8. commit : bu işlem lokaldeki dosyalarımızı depoya gönderilmek üzere hazırlar

VS code da projede bir değişiklik yapıldıktan sonra bunu githuba göndermek için source kontrol tabını açtık Message alanına yapılan değişiklikle alakalı bir mesaj yazdık son olarak commit butonuna basıp gönderilmek üzere hazırladık ve sync yaptık githuba yolladık
